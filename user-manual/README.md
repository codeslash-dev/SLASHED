# SLASHED user manual

End-user documentation for building sites with SLASHED — task-oriented guides,
written for people *using* the framework rather than developing it.

This is different from [`docs/`](../docs), which is the technical/API reference
(token tables, architecture notes) aimed at contributors and tooling. Start here
if you just want to know **what to use, and when**.

## Contents

- [Colors](./colors.md) — the 10 source colors, light/dark, shade ramps, status
  colors, and which token to reach for in common situations.

More pages will be added here as the manual grows (typography, spacing, layout,
theming recipes, etc.).
